# Outlook Thread Defuser

Web版Outlookの長大なメールスレッドで、古い本文や過去の返信履歴をDOMから退避し、ブラウザのフリーズを軽減するMicrosoft Edge用拡張機能です。

## 対象

- `https://outlook.office.com/`
- `https://outlook.office365.com/`
- `https://outlook.live.com/`
- `https://outlook.cloud.microsoft/`

新しいOutlookデスクトップアプリ内には通常のEdge拡張機能を直接追加できないため、対象はWeb版Outlookです。

## インストール

1. ZIPを展開します。
2. Edgeで `edge://extensions/` を開きます。
3. 左側の「開発者モード」をオンにします。
4. 「展開して読み込み」を押します。
5. 展開した `outlook-thread-defuser` フォルダーを選択します。
6. Web版Outlookのタブを再読み込みします。

## 使い方

通常は自動で動作します。拡張機能のアイコンを押すと、以下を変更できます。

- 自動軽量化のオン・オフ
- 表示したまま残す本文数
- 新着メールが上か下かの指定
- 返信履歴を切り離す最小文字数
- DOMノード数のしきい値
- 手動実行と全復元

退避した部分には「この部分を復元」ボタンが表示されます。

## 仕組み

- Outlookの固定CSSクラス名には極力依存せず、ARIA属性とDOM構造から本文を検出します。
- `divRplyFwdMsg`、引用ブロック、メールヘッダーの並びを検出して返信履歴を切り離します。
- 単なる `display:none` ではなく、元のHTMLを文字列として保持し、重いDOMノードを削除します。
- 現在の画面に近い本文を優先して残すため、メールの並び順が異なっても動作しやすくしています。

## プライバシー

- 外部サーバーへの通信処理はありません。
- メール本文を保存領域へ書き込みません。
- 復元用HTMLは、そのOutlookタブが開いている間だけメモリ内に保持します。
- 権限の対象はOutlookの4ドメインとローカル設定のみです。

## 注意点

Outlook WebのDOMはMicrosoft側の更新で変わるため、将来的に検出ロジックの調整が必要になる場合があります。誤って必要な部分が退避された場合は、本文内の「この部分を復元」またはポップアップの「すべて復元」を使用してください。
